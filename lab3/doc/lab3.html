In this lab we will work on object localisation. Object localisation is an application of the histogram backprojection technique we developed in the previous lab. This is a good way to explore the uses of histograms further.

More information on last week's assignment

I'll start with a quick recap of the previous lab. Most of you tried to use the built-in functions that Matlab provides to create a histogram of an image. Since the function hist is not suited for the task at hand (it takes 1-dimensional input), this is not the way to go. You will have to write your own function to compute a histogram from an image.

A histogram is a method of quantising the colours in the image. A colour in the (R,G,B)-space consists of three values R, G and B, and in the histogram, the correlation between these values have to be retained. Therefore, it is not good enough to generate three histograms of the separate colour channels (one for each color channel), but every (R,G,B)-triple will have to be "binned" as a whole. If the histogram would be as accurate as possible, for an (R,G,B)-image with pixel values ranging between 0 and 255, the resulting histogram contain 256x256x256 bins. However, this version is quite sensitive to noise, and huge in memory, so what is generally done is map the colours to a lower-dimensional histogram, e.g. a 16x16x16 histogram. You will have to write a function that determines the appropriate bin for a given color.

One final remark about last week involves the different colour representations you played with in the first lab session. Before creating a histogram, you could choose to convert the image (or the image of your model/object) to other colour representations. This will result in a histogram that has the same properties as the colour representation you used. For instance, converting the (R,G,B)-image to (r,g,b) will result in a representation of your target that is invariant to intensity-differences, since the (r,g,b)-space is invariant to differences in intensity.

Two example histograms can be found here and here. These figures are histograms of the two Nemo-images, created with 16 bins using the normalized-rgb color space.

Also, the previous lab might have some interesting information.

Details on this week's assignment

Object segmentation is transforming an image, a matrix of pixel values, into a collection of objects with properties such as size, shape and position. This is the first step into more meaningful image analysis such as object recognition.

Object Segmentation

The tutorial "Image Processing Toolbox: Example 2 -- Advanced Topics :: Getting Started: Example 2 -- Advanced Topics" in Matlab help shows some techniques to label objects in an image.

The input in that example is a gray scale image, but for our purposes we use the result from the histogram back projection. If all went well, you assigned to every pixel in the image a "probability" that this pixel was part of the target. Hence, this probability-image can be seen as a gray-scale image, in which regions with higher values (e.g. areas that are lighter) should correspond with you target. After going through all the steps in the example, you'll end up with all objects in the image labelled. Hopefully your target is one of the labelled objects. Some morphological operations might be necessary to create a clear object and remove noise. If you do not locate your target in the image, remind the note of last week about weighting the histogram with a histogram of background objects. Also, see if other colour representations can improve your results.

Histogram distances

To check the result you get from the object detection and to maybe choose between several detected objects, you can compare the histograms of the target and the candidate objects. So you create a histogram for the candidate objects in exactly the same fashion as you did for the target. Now you need to compare the candidate and target histogram, in some meaningful way. The reader (available on Blackboard and for sale in paper form as well) discusses several methods.
